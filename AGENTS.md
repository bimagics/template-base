## MCP Connection

The project exposes an MCP server over SSE. Connect any MCP client (Claude Code, Cursor, Windsurf, etc.) using:

- **QA endpoint**: `https://{{PROJECT_ID}}-qa.web.app/api/mcp/sse?key=<MCP_API_KEY>`
- **Production endpoint**: `https://{{PROJECT_ID}}.web.app/api/mcp/sse?key=<MCP_API_KEY>`

### Local MCP Config

Copy `.mcp.json.example` to `.mcp.json` and replace `YOUR_MCP_API_KEY` / `YOUR_CP_API_KEY` with real keys. The `.mcp.json` file is git-ignored.

## Authentication

MCP endpoints require an API key via:
- Query parameter: `?key=<API_KEY>`
- Header: `x-api-key: <API_KEY>`

The key is validated against the `MCP_API_KEY` environment variable. If `MCP_API_KEY` is not set, the MCP server returns 503.

## Deployment Flow

| Branch | Environment | URL |
|--------|-------------|-----|
| `dev` | QA | `https://{{PROJECT_ID}}-qa.web.app` |

Push to `dev` triggers QA deployment. Push to `main` triggers production deployment. The CI/CD pipeline builds a Docker image, pushes to Artifact Registry, deploys to Cloud Run, and updates Firebase Hosting.

## Conventions

- **Routes** go in `src/routes/`
- **Services** (business logic, external integrations) go in `src/services/`

- **DO NOT modify** `.github/workflows/deploy.yml` — managed by WIZBI
- **DO NOT modify** `src/mcp/index.ts` unless changing MCP transport/auth logic

## Getting Started (AI Agents)

1. Run `./setup-local.sh` to set up the local environment
2. Copy `.mcp.json.example` to `.mcp.json` and insert your API keys
3. Connect your MCP client to the QA endpoint for testing
4. Use `npm run dev` for local development with auto-reload
5. Push to `dev` branch to deploy to QA for testing
6. Push to `main` branch to deploy to production
